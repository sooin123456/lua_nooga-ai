# 누가 잘못 AI v2 리브랜딩 설계

## 배경

현재 앱은 `미스터 노우`라는 이름의 Toss WebView 미니앱 MVP다. 사용자는 텍스트, 캡처 OCR, 녹음 흐름, 녹음 파일 흐름으로 싸움 내용을 입력하고, 규칙 기반 분석기로 누가 얼마나 잘못했는지 판정받는다.

v2에서는 앱을 **누가 잘못 AI**로 리브랜딩하고, **미스 노짱**을 앱 안의 판사 캐릭터로 사용한다. 첨부된 애니 판사 캐릭터 느낌을 중심 비주얼로 삼는다. 무료 판독은 기존 규칙 기반 판정을 유지하고, 유료 990원 판독은 추후 서버/AI/판례 검색을 연결할 수 있는 구조와 UX를 먼저 만든다.

- Toss 인앱결제 소개: https://developers-apps-in-toss.toss.im/iap/intro.html

## 목표

- 앱 이름, 메타데이터, 화면 문구를 **누가 잘못 AI**로 바꾼다.
- 캐릭터/브랜드 서브카피로 **미스 노짱이 판독해드립니다**를 사용한다.
- 캐릭터를 하얀 단발머리, 판사 가발, 망치, 무표정한 애니 판사 느낌으로 바꾼다.
- 무료 판독과 유료 990원 판례 판독을 명확히 나눈다.
- 무료 판독 결과 뒤에 이긴 사람이 받고 싶은 것을 입력하고, 유사한 선물/보상 카테고리를 추천한다.
- 토스 쇼핑 연결은 실제 딥링크를 지금 확정하지 않고 `shoppingAdapter`로 분리한다.
- 유료 990원 판례 판독은 실제 결제/서버 없이도 화면 흐름과 어댑터 인터페이스를 먼저 만든다.
- `precedent-kr`는 앱에 통째로 번들하지 않고, 추후 서버 검색 API 또는 선별 인덱스로 연결한다.
- 결과 표현은 재미용 판독이며 법률 자문이 아니라는 안전 문구를 유지한다.

## 비목표

- 이번 v2에서 실제 Toss 인앱결제를 완성하지 않는다.
- 이번 v2에서 판례 12만+건 전체를 앱 번들에 포함하지 않는다.
- 이번 v2에서 실제 Toss Shopping 딥링크 정책을 확정하지 않는다.
- 이번 v2에서 변호사 수준의 법률 판단, 소송 가능성 판단, 배상액 산정, 법률 자문을 제공하지 않는다.
- 참고 캐릭터 이미지를 그대로 복제하지 않는다. 비슷한 분위기의 프로젝트용 캐릭터 자산을 만든다.

## 제품 톤

앱은 장난스럽고 선명한 판정형이다. 다만 판례와 결제 문구가 들어가므로, 법률 서비스처럼 오해되지 않게 안전 문구를 함께 둔다.

주요 카피:

- 앱 이름: **누가 잘못 AI**
- 캐릭터 서브카피: **미스 노짱이 판독해드립니다**
- 홈 헤드라인: **990원 내면 판례까지 뒤져드립니다**
- 보조 문구: **무료로는 가볍게, 유료로는 더 그럴듯하게.**
- 유료 CTA: **990원 판례 판독 열기**
- 무료 CTA: **무료로 먼저 판독하기**
- 안전 문구: **입력된 내용 기준의 재미용 판독이며 법률 자문이 아니에요.**

## 캐릭터 방향

캐릭터는 첨부 이미지의 분위기를 참고한다.

시각 요소:

- 하얀 단발머리
- 판사 가발
- 검은 법복
- 작은 망치
- 무표정하거나 단호한 표정
- 귀엽지만 판결하는 느낌

구현 방식:

- 우선 프로젝트 자산으로 `public/miss-nozzang-judge.png`를 둔다.
- 기본 홈 화면, 결과 화면, 앱 아이콘에서 같은 캐릭터 방향을 사용한다.
- 아이콘은 작은 크기에서도 알아볼 수 있게 얼굴과 망치를 중심으로 단순화한다.

## 무료 판독 흐름

현재 네 입력 흐름은 유지한다.

- 카톡 싸움 붙여넣기
- 증거 캡처 제출하기
- 현장 녹음 시작
- 녹음 파일 불러오기

사용자는 입력 후 텍스트 확인 화면을 거쳐 무료 판독을 실행한다. 무료 판독은 기존 규칙 기반 분석기를 사용한다.

무료 결과 구성:

- 오늘의 판결
- A/B 잘못 퍼센티지
- 근거 3개
- 한 줄 조언
- 이긴 사람 보상 추천 섹션

## 보상 추천 및 토스 쇼핑 UX

결과 화면에서 이긴 사람이 받고 싶은 것을 입력한다.

예시 입력:

- 커피
- 달달한 거
- 사과 선물
- 귀여운 거
- 작은 사치
- 밥 한 끼

클라이언트는 입력 키워드를 선물 카테고리로 매핑한다.

기본 카테고리:

- 커피/음료
- 디저트/간식
- 꽃/감정 회복
- 캐릭터 굿즈
- 생활 소품
- 식사권

추천 결과에는 상품명을 특정하지 않고 카테고리와 검색어를 보여준다. 예를 들어 사용자가 `달달한 거`라고 쓰면 `디저트/간식` 카테고리와 `초콜릿`, `마카롱`, `케이크` 검색어를 추천한다.

CTA:

- **토스 쇼핑에서 비슷한 보상 찾기**

구현 방식:

- `shoppingAdapter`를 만든다.
- MVP에서는 URL을 실제 쇼핑 딥링크로 확정하지 않고, 검색어와 카테고리를 만든 뒤 mock action 또는 외부 URL 어댑터 자리만 둔다.
- 실제 연결 시 Toss `openURL`을 사용하되, 외부 링크 가이드라인을 확인한 뒤 연결한다.

## 유료 990원 판례 판독 흐름

유료 판례 판독은 다음 단계로 구성한다.

1. 사용자가 무료 판독 결과를 본다.
2. 더 그럴듯한 근거를 보고 싶으면 **990원 판례 판독 열기**를 누른다.
3. 현재 v2에서는 실제 결제 대신 결제 준비 화면을 보여준다.
4. 화면에는 `인앱결제 연결 예정`, `판례 검색 서버 연결 예정`, `990원`을 표시한다.
5. 서버 연결 후에는 판례 검색 결과 2~3개와 유사 이유를 보여준다.

결제 정책:

- 무료 판독: 항상 무료
- 유료 판례 판독: 990원
- AI/서버/판례 검색이 필요한 경우에만 유료로 안내

이번 v2 구현:

- `premiumAdapter` 또는 `paymentAdapter` 인터페이스를 만든다.
- 실제 결제 함수는 `notConfigured` 상태를 반환한다.
- UI는 990원 상품 구성을 보여주고, 아직 실결제는 연결되지 않았다는 안내를 제공한다.

## 판례 근거 설계

`precedent-kr`는 약 2.1GB, 12만+ Markdown 파일로 구성되어 있다. 앱에 전체 번들하면 성능과 패키지 크기가 맞지 않는다.

이번 v2 구현:

- `precedentAdapter` 인터페이스를 만든다.
- 클라이언트에는 작은 샘플 데이터 또는 빈 결과 fallback만 둔다.
- 판례 결과 화면 구조를 먼저 만든다.

후속 서버 구현:

- 서버에서 `precedent-kr`를 인덱싱한다.
- 입력 대화에서 키워드와 갈등 유형을 추출한다.
- 관련 판례의 `사건명`, `법원명`, `선고일자`, `판결요지`, `출처`를 2~3개 반환한다.
- 앱은 판례 원문을 단정적으로 해석하지 않고 유사 포인트만 보여준다.

판례 결과 표시:

- 유사 판례 제목
- 법원/선고일자
- 판결요지 일부 요약
- 왜 비슷한지 한 줄
- 출처 링크

안전 문구:

- **판례는 유사한 참고 자료이며, 실제 법률 판단은 사건의 구체적 사실관계에 따라 달라져요.**

- 기존 노란 얼굴 캐릭터 → 애니 판사 캐릭터
- 헤드라인: `990원 내면 판례까지 뒤져드립니다`
- 무료/유료 분리 CTA

텍스트 확인 화면:

- `미스터 노우가 판정` 문구를 `미스 노짱이 판독` 또는 `미스 노짱이 판독해드립니다`로 변경한다.
- 기본 버튼은 `무료 판독 받기`로 바꾼다.

결과 화면:

- `판정 결과` → `판독 결과`
- `오늘의 판정` → `오늘의 판결`
- 무료 결과 아래 보상 추천 섹션 추가
- 유료 판례 판독 CTA 추가

프리미엄 안내 화면 또는 모달:

- 가격: 990원
- 설명: AI/서버/판례 검색 연결 시 제공
- 현재 상태: 결제 연결 준비 중

## 안전 및 검수 정책

- 판례 기반 표현은 법률 자문처럼 보이지 않게 한다.
- `판례 근거로 확정`이 아니라 `유사 판례 참고`라고 표현한다.
- 위험 신호가 있는 경우 기존처럼 안전 안내를 우선한다.
- 위험 신호 결과에서는 토스 쇼핑 보상 추천보다 안전 안내를 우선한다.
- 실제 결제 연결 전에는 결제 버튼이 사용자에게 과금되는 것처럼 보이지 않게 한다.
- 실제 토스 쇼핑 연결 전에는 외부 이동을 명확히 안내한다.

## 테스트 기준

- 홈 화면에 `누가 잘못 AI`, `미스 노짱이 판독해드립니다`, `990원` 문구가 보인다.
- 캐릭터 이미지 또는 대체 캐릭터 UI가 렌더링된다.
- 무료 판독 결과에 `오늘의 판결`과 보상 추천 섹션이 보인다.
- 보상 입력이 카테고리 추천으로 매핑된다.
- 990원 판례 판독 CTA가 결제 준비 화면을 연다.
- 판례 어댑터가 서버 미설정 상태를 안전하게 처리한다.
- 쇼핑 어댑터가 검색어와 카테고리를 만들고 실제 이동은 분리한다.
- 기존 입력 흐름, OCR, 음성 파일 흐름, 안전 신호 테스트가 깨지지 않는다.

## 승인된 결정

- 앱 이름은 **누가 잘못 AI**다.
- 캐릭터/서브브랜드 문구는 **미스 노짱이 판독해드립니다**다.
- 판독 가격은 `100원`이 아니라 **990원**이다.
- 싸움 판독 자체는 무료다.
- AI, 서버, 판례 검색이 필요한 고급 판독만 유료로 안내한다.
- 이긴 사람이 받고 싶은 것을 입력하면 무료 결과에서도 토스 쇼핑 연결용 추천을 제공한다.
- 실제 결제, 서버, 토스 쇼핑 연결은 지금 완성하지 않고 어댑터로 분리한다.
