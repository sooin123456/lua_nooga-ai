# 누가 잘못 AI

누가 잘못 AI는 Toss Apps in Toss WebView 환경에서 동작하는 미니 앱입니다. 루아 AI가 카카오톡이나 일반 채팅에서 가져온 갈등 상황을 가볍게 판독해 주는 재미용 서비스입니다.

## 주요 기능

- 카카오톡/채팅 텍스트를 입력해 갈등 내용을 판독합니다.
- 대화 스크린샷을 올리고 OCR로 추출한 내용을 확인한 뒤 판독합니다.
- 녹음 흐름에서 사용자가 대화 내용을 텍스트로 확인하거나 직접 입력한 뒤 판독합니다.
- 오디오 파일 흐름에서도 분석 전에 텍스트 확인 또는 수동 입력을 거칩니다.
- 실시간 판정방을 만들면 닉네임으로 입장한 뒤 초대 링크로 상대를 부릅니다. A/B 당사자가 모두 들어오면 60초 카운트가 시작되고, 세 번째 입장자부터는 읽기 전용 관전자로 들어옵니다. 판정 뒤에는 대화 원문을 삭제하고 결과만 남기는 흐름을 제공합니다.
- 메인 화면의 오늘의 핫 Battle에서 공유된 판독 순위를 별도 페이지로 열고, 상세 화면에서 대화 요약과 댓글을 확인할 수 있습니다. 서버 연결 후에는 공유 판독 결과와 댓글/좋아요 데이터를 붙일 수 있는 형태입니다.
- 무료 판독 결과에서 이긴 사람이 받고 싶은 보상을 입력하면 Toss Shopping 연결용 추천 카테고리와 검색어를 보여줍니다.
- 판독 결과는 카톡, 텔레그램, 링크 공유로 사람들에게 물어볼 수 있습니다. Supabase가 연결된 환경에서는 원문 대화가 아니라 판독 결과 JSON만 공유 저장하고, 핫 Battle 상세에서 댓글/좋아요를 서버에 저장합니다.
- 홈과 결과 화면은 레드 리포트 모션 UI를 사용하며, 사용자의 `prefers-reduced-motion` 설정을 존중합니다.

## 가격/연동 정책

- 싸움 판독 자체는 하루 3회 무료이며, 공유 결과 생성에 성공하면 하루 1회를 추가로 받을 수 있습니다.
- 상대방이 무료 판독에 이의를 제기하면 990원을 더 내고 판례 근거로 다시 따져보는 상품으로 연결합니다.
- Toss 인앱결제는 `precedent-verdict-990` SKU를 사용하도록 구성되어 있습니다.
- 판례 검색 API는 Vercel `/api/precedents/search` 또는 별도 판례 API 서버를 통해 호출할 수 있습니다.
- Toss Shopping 연결은 설정된 검색 URL을 `openURL`로 여는 방식입니다.

### 무료 AI 판독

무료 판독은 `/api/ai/free-judgment`에서 서버 AI로 실행됩니다. 프론트는 익명 사용자 키와 대화 텍스트, 사용자 관점을 보내고, 서버는 하루 3회 무료 제한을 확인한 뒤 AI JSON 판정 결과를 반환합니다. 서버 연결이 실패하면 앱은 로컬 규칙 기반 판독으로 임시 fallback합니다.

## 판례 데이터

`legalize-kr/precedent-kr`는 서버 인덱싱 대상으로 보고, 클라이언트 번들에 통째로 포함하지 않습니다. 앱은 서버가 반환하는 유사 판례 제목, 법원/선고일자, 요약, 유사 이유, 출처 링크를 표시합니다.

## 실행

```bash
npm install
npm run dev
```

## 검증

```bash
npm run lint
npm test
npm run build
```

`@apps-in-toss/ait-format@1.0.0`은 Node.js 24 이상을 요구합니다. 로컬 Node.js 버전이 낮으면 빌드가 실패할 수 있으니, 필요한 경우 Node.js 24 이상에서 다시 검증해 주세요. 현재 환경에서 빌드가 통과하더라도 패키지 경고가 있다면 이 기준을 유지합니다.

`npm run build`는 현재 통과하지만 OCR/Tesseract 기능으로 번들 크기가 커져 Vite의 `Some chunks are larger than 500 kB after minification` 경고가 표시될 수 있습니다. 이 메시지는 빌드 실패가 아니라 번들 크기 경고입니다.

## 개인정보 원칙

- 일반 무료 판독에서는 사용자가 입력한 텍스트, 이미지, 오디오를 서버에 저장하지 않습니다.
- 실시간 판정방 참여자 닉네임과 메시지는 Supabase에 임시 저장되며, 메시지는 판정 또는 만료 후 삭제됩니다. 세 번째 이후 입장자는 관전자라 메시지를 보낼 수 없습니다.
- 공유 결과방은 판독 결과와 핫 Battle 반응만 저장하며, 사용자가 입력한 원문 대화는 저장하지 않습니다.
- 핫 Battle 공개 댓글은 위협 표현, 과도한 통제 표현, 연락처/이메일 같은 개인정보를 차단합니다.
- 990원 이의제기 판례 분석을 실행하면 입력 대화 텍스트가 판례 검색 서버로 전송됩니다.
- 스크린샷 OCR은 기본적으로 브라우저에서 처리합니다.
- 녹음 및 오디오 파일은 분석 전에 사용자의 텍스트 확인 또는 수동 입력을 필요로 합니다.
- 결과는 입력된 내용 기준의 재미용 판독이며 법률, 의료, 심리 상담 또는 전문적 판단을 대신하지 않습니다.
- 판례는 유사한 참고 자료이며, 실제 법률 판단은 사건의 구체적 사실관계에 따라 달라집니다.
